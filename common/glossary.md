## 1. Overview

This glossary defines common terms used across CodeFleet Labs projects and documentation.

---

## 2. General Terms

| Term             | Definition                                         |
| ---------------- | -------------------------------------------------- |
| Application      | A software system built for a specific purpose     |
| Module           | A self-contained component within an application   |
| Service          | Business logic layer responsible for operations    |
| API              | Application Programming Interface                  |
| Endpoint         | A URL where an API can be accessed                 |
| Deployment       | Process of releasing software to a server          |
| Environment      | Runtime configuration (local, staging, production) |
| Runbook          | Operational procedures for maintaining systems     |
| ADR              | Architecture Decision Record                       |
| CI/CD            | Continuous Integration / Continuous Deployment     |
| Monolith         | Single deployable application                      |
| Modular Monolith | Monolith divided into independent modules          |
| Microservice     | Independent deployable service                     |
| Reverse Proxy    | Server that forwards requests to backend servers   |
| Worker           | Background process handling tasks                  |
| Queue            | Message queue for background jobs                  |
| Cache            | Temporary storage for fast access                  |
| Broker           | Message transport system (e.g., Redis)             |

---

## 3. Django / Backend Terms

| Term               | Definition                           |
| ------------------ | ------------------------------------ |
| Model              | Database representation in Django    |
| Serializer         | Converts model data to JSON and back |
| View               | Handles HTTP requests                |
| ViewSet            | DRF view handling multiple actions   |
| Migration          | Database schema change               |
| ORM                | Object Relational Mapper             |
| Middleware         | Request/response processing layer    |
| Signal             | Event triggered on model actions     |
| Management Command | Custom Django CLI command            |
